# S-Protect-PY 领域模型词汇表

## 核心概念

| 术语 | 定义 |
|------|------|
| **项目** | 一组 Python 源文件构成的软件工程，是加密/混淆的最小工作单元 |
| **入口文件** | 项目中被用户直接执行（`python xxx.py`）的文件，配置在 JSON5 中 |
| **加密项目** | 经过 S-Protect 处理后的项目产物，包含混淆代码和运行时启动器 |
| **启动器** | 内嵌到每个加密项目中的 runtime 模块，负责解密、校验、反调试和加载执行 |

## 加密与混淆

| 术语 | 定义 |
|------|------|
| **混淆** | 改变代码形态但不改变语义的操作（变量重命名、控制流平坦化等） |
| **加密** | 将代码转换为密文，需要对应密钥才能还原为可执行形态 |
| **运行时解密** | 启动器在内存中解密代码后执行，磁盘始终保持加密态 |
| **全链路加密** | 链式依赖 + 全联通混合的多文件联合加密策略 |
| **密钥分片** | 将私钥切割为多片，嵌入不同文件，需全部收集才能还原 |
| **真假索引** | 分片位置索引包含真/假条目，真索引内也混有假偏移，防止逆向分析 |

## 完整性保护

| 术语 | 定义 |
|------|------|
| **校验环** | 每个文件内嵌其他文件的哈希值，形成环形校验网络 |
| **交叉签名** | 文件 A 签名文件 B，文件 B 签名文件 C，形成签名链 |
| **自校验** | 启动器校验自身完整性，比对内嵌哈希 |
| **环境指纹** | 绑定到目录结构、环境变量、用户名的运行时环境标识 |
| **防回滚** | 通过加密时间戳防止攻击者用旧版本替换新版本 |

## 反逆向

| 术语 | 定义 |
|------|------|
| **反调试** | 检测 pdb/ptrace/调试器附加，发现后执行自毁或退出 |
| **内存保护** | 反调试触发时清空敏感内存并退出 |
| **代码虚拟化** | 将 Python 字节码翻译为自定义虚拟指令集，用内置解释器执行 |

## 运行时机制

| 术语 | 定义 |
|------|------|
| **启动器常驻** | 运行时解密流程全程由启动器进程控制，不暴露密钥 |
| **隐蔽目录** | 存放加密态文件的专用目录（如 `_runtime/`），内容为乱码 |
| **导入钩子** | 自定义 `sys.meta_path` 钩子，拦截 `.pye` 后缀的模块导入并解密 |
| **过期控制** | 内嵌加密时间和过期时间，联合系统时间和 NTP 校验执行期 |
| **水印** | 注入到文件级/代码级/运行时级的隐形标识，用于追踪泄露源 |

## CLI 与配置

| 术语 | 定义 |
|------|------|
| **JSON5 配置** | 使用 JSON5 格式的配置文件，支持注释和灵活语法 |
| **加密策略** | 配置中定义的文件依赖模式（链式/全联通/混合） |
| **规则系统** | 用户自定义的混淆/加密规则，包括命名规则、保留规则、排除规则 |
